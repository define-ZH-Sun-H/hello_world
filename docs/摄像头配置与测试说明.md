# OV2640 摄像头配置与测试说明

> 硬件：普中 ESP32-S3 开发板（ESP32-S3-N16R8，16MB Flash + 8MB PSRAM）
> 框架：ESP-IDF v5.5.1
> 摄像头：OV2640（DVP 并行接口，JPEG 输出）
> 日期：2026-05-28

---

## 一、背景

摄像头 OV2640 输出 JPEG 格式图片，每帧约 20~50KB（QVGA 分辨率）。ESP32-S3 的内部 DRAM 不足以存放完整的 JPEG 帧缓冲区，**必须启用外部 PSRAM**。

另外，开发板的引脚是复用的，摄像头会占用某些原本被其他外设使用的 GPIO。测试摄像头的需要临时断开这些外设。

---

## 二、引脚映射

从普中开发板官方资料（Arduino 示例）提取的 OV2640 接线定义：

| 摄像头信号 | GPIO | 说明 |
|-----------|:----:|------|
| XCLK | 15 | 主时钟输出 |
| SIOD | 4 | SCCB 数据线 (I2C SDA) |
| SIOC | 5 | SCCB 时钟线 (I2C SCL) |
| Y2 (D7) | 11 | 像素数据位 2 |
| Y3 (D6) | 9 | 像素数据位 3 |
| Y4 (D5) | 8 | 像素数据位 4 |
| Y5 (D4) | 10 | 像素数据位 5 |
| Y6 (D3) | 12 | 像素数据位 6 |
| Y7 (D2) | 18 | 像素数据位 7 |
| Y8 (D1) | 17 | 像素数据位 8 |
| **Y9 (D0)** | **16** | **像素数据位 9 — 与 WS2812 RGB 冲突** |
| VSYNC | 6 | 帧同步 |
| HREF | 7 | 行同步 |
| **PCLK** | **13** | **像素时钟 — 与 DS18B20 冲突** |
| PWDN | -1 | 未使用（硬件拉低） |
| RESET | -1 | 未使用（硬件拉高） |

### GPIO 冲突表

| GPIO | 摄像头 | 冲突外设 | 冲突原因 |
|:----:|--------|---------|---------|
| 13 | PCLK | DS18B20 (DQ) | 摄像头运行时 PCLK 是连续脉冲，DS18B20 的 1-Wire 时序会被干扰 |
| 16 | Y9 (D0) | WS2812 RGB (DATA) | DVP 并行数据总线位，RGB 的 RMT 信号会干扰像素数据 |

**测试摄像头时必须关闭的外设：** DS18B20、WS2812 RGB。

## 三、操作步骤

### Step 1: 启用 PSRAM

修改 `sdkconfig`，添加以下配置：

```
CONFIG_SPIRAM=y                          # 启用 PSRAM
CONFIG_SPIRAM_MODE_OCT=y                 # 8线 Octal 模式
CONFIG_SPIRAM_TYPE_AUTO=y                # 自动检测型号
CONFIG_SPIRAM_SPEED_80M=y                # 80MHz 时钟
CONFIG_SPIRAM_USE_MALLOC=y               # malloc 可使用 PSRAM
CONFIG_SPIRAM_BOOT_HW_INIT=y             # 启动时初始化硬件
CONFIG_SPIRAM_BOOT_INIT=y                # 启动时完成初始化
CONFIG_SPIRAM_IGNORE_NOTFOUND=y          # 无 PSRAM 时不崩溃
```

由于项目启用了 `MINIMAL_BUILD`，需在 `main/CMakeLists.txt` 的 `PRIV_REQUIRES` 中添加 `esp_psram` 才能将 PSRAM 组件包含到构建中。

添加后执行 `idf.py reconfigure` 让 Kconfig 补全依赖项。

### Step 2: 添加摄像头组件

通过 ESP-IDF 组件管理器安装：

```bash
idf.py add-dependency "espressif/esp32-camera"
```

这会自动：
- 创建 `main/idf_component.yml` 记录依赖
- 下载 `esp32-camera` 到 `managed_components/espressif__esp32-camera/`
- 同时下载依赖 `esp_jpeg`

执行 `idf.py reconfigure` 确认组件加载成功。

### Step 3: 关闭冲突外设

修改 `main/main.c`，注释掉与摄像头共用 GPIO 的外设初始化：

```c
// sensor_init();   /* DS18B20 用 GPIO13，与摄像头 PCLK 冲突 */
// rgb_init();      /* WS2812 用 GPIO16，与摄像头 Y9 冲突 */
```

同时注释掉 `create_all_tasks()` 中的 `sensor_task` 创建，防止任务尝试操作已被摄像头占用的 GPIO。

### Step 4: 编写摄像头测试代码

新增两个文件：

**`main/camera_test.h`** — 声明测试接口：

```c
esp_err_t camera_test_init(void);           // 初始化摄像头
esp_err_t camera_test_capture(const char *save_path);  // 拍照并存文件
```

**`main/camera_test.c`** — 主要逻辑：

1. 按开发板引脚映射填充 `camera_config_t` 结构体
2. 关键配置：
   - `pixel_format = PIXFORMAT_JPEG` — JPEG 直接输出
   - `frame_size = FRAMESIZE_QVGA` — 320x240，先跑通再提高
   - `fb_location = CAMERA_FB_IN_PSRAM` — 帧缓冲区放在 PSRAM
   - `fb_count = 1` — 单缓冲区，省内存
3. `camera_test_init()` 调用 `esp_camera_init()` 并打印传感器信息
4. `camera_test_capture()` 调用 `esp_camera_fb_get()` 获取一帧，写入 SD 卡

更新 `main/CMakeLists.txt`，在 `SRCS` 中添加 `camera_test.c`。

### Step 5: 编译

```bash
idf.py build
```

成功输出：
```
hello_world.bin binary size 0x75d60 bytes. Smallest app partition is 0x200000 bytes. 0x18a2a0 bytes (77%) free.
```

## 四、烧录与测试

### 烧录

```bash
idf.py -p /dev/ttyUSB0 flash monitor
```

（将 `/dev/ttyUSB0` 替换为实际端口号）

### 预期串口输出

```
I (xxx) camera: 摄像头初始化完成 (QVGA 320x240 JPEG)
I (xxx) camera: 拍照成功: 320x240, 18432 bytes, 格式=4
I (xxx) camera: 已保存: /sdcard/test.jpg (18432 bytes)
```

### 验证步骤

1. 烧录后观察串口日志，确认摄像头初始化成功
2. 看到 "拍照成功" 和 "已保存" 日志
3. 拔出 SD 卡，插入电脑确认 `test.jpg` 文件存在且可打开
4. 图片应为 320x240 的 JPEG，内容为摄像头视野

## 五、恢复正常使用（非摄像头模式）

测试完成后，如需使用 DS18B20 和 WS2812 RGB，恢复以下改动：

1. `main/main.c`：
   - 取消注释 `sensor_init();` 和 `rgb_init();`
   - 取消注释 `sensor_task` 的 `xTaskCreatePinnedToCore()`
   - 注释掉或删除摄像头测试代码块（`camera_test_init()` / `camera_test_capture()`）

2. 也可保留摄像头代码，通过按键条件编译选择启用哪个外设模式。

---

## 六、注意事项

1. **PSRAM 是必须的** — OV2640 的 JPEG 帧缓冲区无法放入内部 DRAM，不启用 PSRAM 摄像头无法工作
2. **帧缓冲区必须在 PSRAM** — `fb_location = CAMERA_FB_IN_PSRAM` 不能省略
3. **首次运行记得格式化** — `format_if_mount_failed = true` 确保 SPIFFS 首次挂载成功
4. **SD 卡需要 FAT32 格式** — 否则 `esp_vfs_fat_sdspi_mount()` 会挂载失败
5. **摄像头排线要插紧** — DVP 并行接口的 24 根信号线，接触不良会出图异常（花屏/绿屏）
