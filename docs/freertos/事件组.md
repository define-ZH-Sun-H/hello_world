# FreeRTOS 事件组（Event Group）使用指南

## 什么是事件组

事件组是一个 **位掩码**（bitmask）同步对象，每个 bit 代表一个独立的事件标志。任务可以：

- **等待** 一个或多个 bit 同时被置位
- **设置**（置 1）或 **清除**（清 0）任意 bit
- 选择 **等待所有**（AND）或 **等待任意一个**（OR）

```
Bit7  Bit6  Bit5  Bit4  Bit3  Bit2  Bit1  Bit0
  0     0     0     0     0     0     0     0     ← 初始全 0
                            ↓ 某任务设置 bit3
  0     0     0     0     1     0     0     0     ← bit3=1
                            ↓ 等待 bit3 的任务被唤醒
```

### 与队列、信号量的对比

| 同步方式 | 传递什么 | 适合场景 |
|---------|---------|---------|
| **队列** | 数据（任意结构体） | ISR 向任务传递数据 |
| **二值信号量** | "发生了"（无数据） | 简单唤醒，一次一事件 |
| **互斥锁** | 访问权 | 保护共享资源 |
| **事件组** | 多个事件的组合状态 | 一个任务等 **多个条件** 同时满足 |

## API 总览

```c
#include "freertos/FreeRTOS.h"
#include "freertos/event_groups.h"

/* 创建 */
EventGroupHandle_t xEventGroupCreate(void);

/* 任务级操作 */
EventBits_t xEventGroupSetBits(EventGroupHandle_t eg, EventBits_t bits);
EventBits_t xEventGroupClearBits(EventGroupHandle_t eg, EventBits_t bits);
EventBits_t xEventGroupGetBits(EventGroupHandle_t eg);

/* 等待（可阻塞） */
EventBits_t xEventGroupWaitBits(
    EventGroupHandle_t eg,
    EventBits_t bitsToWaitFor,    /* 等哪些 bit */
    BaseType_t xClearOnExit,      /* 满足后自动清除 */
    BaseType_t xWaitForAllBits,   /* pdTRUE = AND, pdFALSE = OR */
    TickType_t xTicksToWait       /* 超时 */
);

/* ISR 安全版 */
BaseType_t xEventGroupSetBitsFromISR(
    EventGroupHandle_t eg,
    EventBits_t bits,
    BaseType_t *pxHigherPriorityTaskWoken
);
```

## 使用模式

### 模式一：多任务等待同一事件（广播）

一个任务负责"生产"事件，多个任务同时等待：

```c
EventGroupHandle_t xEventGroup;

/* 任务 A：等待数据就绪 + 网络就绪后开始工作 */
void task_a(void *pv)
{
    /* 等 bit0 AND bit1 同时置位 */
    xEventGroupWaitBits(xEventGroup,
        BIT0 | BIT1,       /* 等 bit0 和 bit1 */
        pdTRUE,            /* 满足后自动清除 */
        pdTRUE,            /* AND：两个都要等 */
        portMAX_DELAY);

    printf("Task A: 数据 + 网络已就绪，开始\n");
    vTaskDelete(NULL);
}

/* 任务 B：只等网络就绪 */
void task_b(void *pv)
{
    xEventGroupWaitBits(xEventGroup, BIT1,
        pdTRUE, pdFALSE, portMAX_DELAY);

    printf("Task B: 网络已就绪\n");
    vTaskDelete(NULL);
}

/* 初始化任务：10 秒内逐步产生事件 */
void init_task(void *pv)
{
    vTaskDelay(pdMS_TO_TICKS(3000));
    printf("数据采集完成\n");
    xEventGroupSetBits(xEventGroup, BIT0);

    vTaskDelay(pdMS_TO_TICKS(2000));
    printf("网络连接完成\n");
    xEventGroupSetBits(xEventGroup, BIT1);

    vTaskDelete(NULL);
}

void app_main(void)
{
    xEventGroup = xEventGroupCreate();
    xTaskCreate(init_task, "init", 2048, NULL, 3, NULL);
    xTaskCreate(task_a, "taskA", 2048, NULL, 2, NULL);
    xTaskCreate(task_b, "taskB", 2048, NULL, 2, NULL);
}
```

**执行结果：**
```
Task A: 数据 + 网络已就绪，开始
Task B: 网络已就绪
（约 5 秒后两个任务同时被唤醒）
```

### 模式二：OR 等待（任一事件唤醒）

适合"任何一个条件满足就执行"的场景：

```c
#define FLAG_BUTTON   BIT0   /* 按键按下 */
#define FLAG_TIMEOUT  BIT1   /* 定时超时 */
#define FLAG_SENSOR   BIT2   /* 传感器触发 */

void monitor_task(void *pv)
{
    while (1) {
        EventBits_t bits = xEventGroupWaitBits(
            monitor_eg,
            FLAG_BUTTON | FLAG_TIMEOUT | FLAG_SENSOR,
            pdTRUE,     /* 清除所有已满足的 bit */
            pdFALSE,    /* OR：任一满足即唤醒 */
            pdMS_TO_TICKS(5000)); /* 最多等 5 秒 */

        if (bits & FLAG_BUTTON)
            printf("按键触发\n");
        if (bits & FLAG_TIMEOUT)
            printf("超时触发（5 秒无事件）\n");
        if (bits & FLAG_SENSOR)
            printf("传感器触发\n");
    }
}
```

### 模式三：ISR 置位，任务处理

```c
EventGroupHandle_t sensor_eg;
#define EVT_TEMP_READY   BIT0   /* 温度读取完成 */
#define EVT_HUMI_READY   BIT1   /* 湿度读取完成 */
#define EVT_ERROR        BIT7   /* 传感器错误 */

/* ISR 中安全置位 */
void IRAM_ATTR sensor_isr_handler(void *arg)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    if (/* 温度转换完成 */) {
        xEventGroupSetBitsFromISR(sensor_eg, EVT_TEMP_READY,
                                  &xHigherPriorityTaskWoken);
    }
    if (/* 错误标志 */) {
        xEventGroupSetBitsFromISR(sensor_eg, EVT_ERROR,
                                  &xHigherPriorityTaskWoken);
    }

    /* 如果有高优先级任务被唤醒，立即切换 */
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

/* 任务等温度+湿度都就绪后才处理 */
void process_task(void *pv)
{
    while (1) {
        EventBits_t bits = xEventGroupWaitBits(
            sensor_eg,
            EVT_TEMP_READY | EVT_HUMI_READY,  /* 等温度+湿度都完成 */
            pdTRUE,                            /* 自动清除 */
            pdTRUE,                            /* AND */
            portMAX_DELAY);

        if (bits & EVT_ERROR) {
            printf("传感器错误，跳过\n");
            continue;
        }

        printf("温度+湿度就绪，开始处理\n");
    }
}
```

## 在 ESP32-S3 项目中的应用

### 实例：启动条件同步

设备启动时需要完成多个前置条件才能进入主循环：

```c
/* freertos/event_groups.h 已内置 BIT0~BIT31 宏 */
#define START_WIFI_OK    BIT0     /* WiFi 连接成功 */
#define START_SENSOR_OK  BIT1     /* 传感器初始化完成 */
#define START_OLED_OK    BIT2     /* OLED 显示就绪 */
#define START_ALL        (START_WIFI_OK | START_SENSOR_OK | START_OLED_OK)

EventGroupHandle_t start_eg;

void app_main(void)
{
    start_eg = xEventGroupCreate();

    /* 创建各个初始化任务 */
    xTaskCreate(wifi_init_task,   "wifi_init",   4096, NULL, 4, NULL);
    xTaskCreate(sensor_init_task, "sensor_init", 2048, NULL, 4, NULL);

    /* 主任务等待所有初始化完成 */
    xTaskCreate(main_task, "main", 4096, NULL, 3, NULL);
}

void wifi_init_task(void *pv)
{
    /* ... 初始化 WiFi ... */
    printf("WiFi 初始化完成\n");
    xEventGroupSetBits(start_eg, START_WIFI_OK);
    vTaskDelete(NULL);
}

void sensor_init_task(void *pv)
{
    ds18b20_init();
    dht11_init();
    printf("传感器初始化完成\n");
    xEventGroupSetBits(start_eg, START_SENSOR_OK);
    vTaskDelete(NULL);
}

void main_task(void *pv)
{
    /* 阻塞等待三个条件全部满足 */
    xEventGroupWaitBits(start_eg, START_ALL,
                        pdTRUE,        /* 清除所有 bit */
                        pdTRUE,        /* AND：全部就绪 */
                        pdMS_TO_TICKS(30000)); /* 30 秒超时 */

    printf("所有初始化完成，进入主循环\n");

    while (1) {
        /* 主循环逻辑 */
        vTaskDelay(100);
    }
}
```

### 实例：带超时的传感器轮询

```c
#define DATA_WIFI      BIT0
#define DATA_DS18B20   BIT1
#define DATA_DHT11     BIT2
#define DATA_ALL       (DATA_WIFI | DATA_DS18B20 | DATA_DHT11)

EventGroupHandle_t data_eg;

void sensor_poll_task(void *pv)
{
    while (1) {
        /* 触发一次采集 */
        ds18b20_start();          /* 启动 DS18B20 转换 */
        /* DHT11 读取一次 */
        uint8_t t, h;
        dht11_read_data(&t, &h);

        /* 标记数据就绪 */
        xEventGroupSetBits(data_eg, DATA_DS18B20 | DATA_DHT11);

        /* 等所有数据收集齐（含 WiFi 数据，带 10 秒超时） */
        EventBits_t bits = xEventGroupWaitBits(
            data_eg, DATA_ALL,
            pdTRUE, pdTRUE,
            pdMS_TO_TICKS(10000));

        if ((bits & DATA_ALL) == DATA_ALL) {
            printf("全部数据就绪，刷新显示\n");
            /* 刷新 OLED */
            g_disp.ds18b20_temp = ds18b20_get_temperature();
            g_disp.dht11_temp = t;
            g_disp.dht11_humi = h;
            g_disp.dirty = true;
        } else {
            printf("数据收集超时（缺失: 0x%02X）\n",
                   DATA_ALL ^ (bits & DATA_ALL));
        }

        vTaskDelay(pdMS_TO_TICKS(5000));  /* 5 秒一轮 */
    }
}
```

## 注意事项

### 1. 可用 bit 数

ESP32-S3 上 `EventBits_t` 是 **32 位**，但实际使用中：

| 位 | 可用性 |
|----|--------|
| Bit0 ~ Bit23 | **可用**（共 24 bits） |
| Bit24 ~ Bit31 | **保留给内核**，不要使用 |

### 2. ISR 中只能用 `SetBitsFromISR`

```c
/* ❌ 错误：ISR 中不能直接调用 */
void IRAM_ATTR isr(void *arg) {
    xEventGroupSetBits(eg, BIT0);          // 不可在 ISR 使用
}

/* ✅ 正确 */
void IRAM_ATTR isr(void *arg) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xEventGroupSetBitsFromISR(eg, BIT0, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

### 3. `xClearOnExit` 的作用

```c
/* pdTRUE：wait 返回后自动清除等待的 bit（适合一次性事件） */
xEventGroupWaitBits(eg, BIT0 | BIT1, pdTRUE /*clear*/, pdTRUE, portMAX_DELAY);
/* 返回后 BIT0 和 BIT1 自动清 0 */

/* pdFALSE：不自动清除（适合电平型状态） */
xEventGroupWaitBits(eg, BIT0, pdFALSE /*不清理*/, pdFALSE, 0);
/* 返回后 BIT0 保持不变，需要手动 xEventGroupClearBits() */
```

### 4. 超时返回值判断

`xEventGroupWaitBits` 返回的是 **满足条件时的实际 bit 值**。超时返回 0，需要区分：

```c
EventBits_t bits = xEventGroupWaitBits(eg, BIT0 | BIT1,
                                       pdTRUE, pdTRUE,
                                       pdMS_TO_TICKS(100));

if (bits == 0) {
    printf("超时：bit0 和 bit1 未同时就绪\n");
} else if ((bits & (BIT0 | BIT1)) == (BIT0 | BIT1)) {
    printf("成功：两个条件都满足\n");
}
```

### 5. 事件组 vs 多个二值信号量

| 场景 | 事件组 | 多个二值信号量 |
|------|--------|--------------|
| 等 **多个** 事件都发生 | ✅ 一个 `WaitBits` 搞定 | ❌ 需要逐个 Take |
| 等 **任意一个** 事件 | ✅ OR 模式 | ✅ 需 Selec 机制 |
| ISR 传递数据 | ❌（只传 bit，不传数据） | ❌ |
| 有初始状态需求 | ✅ 可预设 bit 值 | ❌ |

### 6. 事件组创建位置

事件组应在任务创建之前创建，避免竞态：

```c
void app_main(void)
{
    /* ✅ 先创建事件组 */
    EventGroupHandle_t eg = xEventGroupCreate();

    /* ✅ 再创建可能立即使用事件的 ISR/任务 */
    gpio_isr_handler_add(IO_PIN, isr_handler, NULL);
    xTaskCreate(worker_task, "worker", 2048, eg, 5, NULL);
    /*                 传递句柄给任务 ↑↑ */
}
```

## 总结

| 要点 | 说明 |
|------|------|
| **本质** | 32 位位掩码，每个 bit 一个事件 |
| **AND 模式** | `xWaitForAllBits = pdTRUE`：等所有 bit 同时置位 |
| **OR 模式** | `xWaitForAllBits = pdFALSE`：任一 bit 置位即唤醒 |
| **自动清除** | `xClearOnExit = pdTRUE`：一次性触发 |
| **手动清除** | `xClearOnExit = pdFALSE`：电平型状态 |
| **ISR 安全** | 用 `xEventGroupSetBitsFromISR` |
| **保留位** | 不要用 Bit24~Bit31 |
| **典型用途** | 等待多条件初始化完成、多传感器数据就绪、广播通知 |
