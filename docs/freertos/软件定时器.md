# FreeRTOS 软件定时器使用指南

## 什么是软件定时器

FreeRTOS 软件定时器是**基于 tick 计数**的定时机制。创建后由 **定时器服务任务**（`tmr Svc`）统一管理，定时到达时调用回调函数。

```
app_main                     tmr Svc 任务
  │                              │
  ├── xTimerCreate()             │     ┌──────────────────┐
  ├── xTimerStart()  ──────────► │ ──► │ 定时器命令队列    │
  │                              │     │  start / stop    │
  │                              │     │  reset / change  │
  │                              │     └──────────────────┘
  │                              │            │
  │                              │   定时到达 ─┤
  │                              │     ┌──────┴──────┐
  │                              │     │  回调函数    │
  │                              │     └─────────────┘
```

### 硬件定时器 vs 软件定时器

| 类型 | 精度 | 资源 | 回调上下文 | 适用场景 |
|------|------|------|-----------|---------|
| **硬件定时器** | 微秒级 | 芯片硬件资源（有限） | ISR 上下文 | PWM、输入捕获、精确计时 |
| **软件定时器** | tick 级（10ms） | 无硬件占用 | `tmr Svc` 任务上下文 | 超时管理、周期性任务 |

> 软件定时器的精度受 tick 周期限制。100Hz tick 下，最短定时周期为 10ms（`pdMS_TO_TICKS(10)`）。

### `tmr Svc` — 定时器服务任务

所有软件定时器的回调都在**同一个**系统任务 `tmr Svc` 中运行：

```
首次 xTimerCreate() 时 ──► 内核自动创建 tmr Svc 任务
                              │
                    ┌─────────┴──────────┐
                    │ 优先级: 1           │  ← configTIMER_TASK_PRIORITY（默认 1）
                    │ 栈: configMINIMAL   │
                    │                     │
                    │ 工作循环:           │
                    │   1. 等命令队列      │  ← 接收 Start/Stop/Reset 请求
                    │   2. 检查定时器超时  │  ← 到期的调用回调
                    │   3. 重复            │
                    └─────────────────────┘
```

**关键性质：**

| 性质 | 说明 |
|------|------|
| **创建时机** | 调用 `xTimerCreate()` 时自动创建（如果尚未存在） |
| **任务名** | `tmr Svc`（可在 `vTaskList` 中看到） |
| **优先级** | `configTIMER_TASK_PRIORITY`，ESP-IDF 默认 **1**（仅高于 IDLE） |
| **回调所在任务** | **所有**定时器回调都在这个任务里跑 |
| **数量** | 整个系统只有一个 `tmr Svc` 任务 |
| **可配置项** | 优先级、栈大小、命令队列长度（均在 sdkconfig 中） |

**通俗理解：** `tmr Svc` 就像一个共享秘书。你（你的任务）告诉秘书"10 分钟后叫我"，到时间了秘书喊你。但秘书只有一个，如果他正在喊别人（执行别人的回调），你的定时器就得等着。

## 创建定时器

### `xTimerCreate()`

```c
TimerHandle_t xTimerCreate(
    const char * const       pcName,              // 定时器名（调试用）
    const TickType_t         xTimerPeriod,        // 周期 tick 数
    const UBaseType_t        uxAutoReload,        // pdTRUE=重载, pdFALSE=一次性
    void * const             pvTimerID,           // 用户数据指针
    TimerCallbackFunction_t  pxCallbackFunction    // 回调函数
);
```

- **返回：** 定时器句柄，内存不足返回 NULL
- 定时器创建后**不立即运行**，需要调用 `xTimerStart()` 启动

### 回调函数

```c
void vTimerCallback(TimerHandle_t xTimer);
```

回调通过 `pvTimerGetID()` 获取创建时传入的用户数据：

```c
typedef struct {
    int counter;
    const char *name;
} timer_user_data_t;

timer_user_data_t my_data = { .counter = 0, .name = "sensor_timer" };

void my_callback(TimerHandle_t xTimer)
{
    timer_user_data_t *data = (timer_user_data_t *)pvTimerGetID(xTimer);
    data->counter++;
    printf("%s: 第 %d 次触发\n", data->name, data->counter);
}

void app_main(void)
{
    TimerHandle_t xTimer = xTimerCreate("sensor",
                                        pdMS_TO_TICKS(1000),
                                        pdTRUE,               // 自动重载
                                        &my_data,
                                        my_callback);
    xTimerStart(xTimer, 0);
}
```

## 启停控制

### `xTimerStart()` — 启动

```c
BaseType_t xTimerStart(TimerHandle_t xTimer, TickType_t xTicksToWait);
```

- 启动定时器，经过**一个完整周期**后才首次触发
- `xTicksToWait`：等待定时器命令队列的**超时时间**
- **返回：** `pdPASS` 或 `pdFALSE`

### `xTimerStop()` — 停止

```c
BaseType_t xTimerStop(TimerHandle_t xTimer, TickType_t xTicksToWait);
```

- 停止正在运行的定时器
- 已停止的定时器可以再次 `Start`

### `xTimerReset()` — 复位

```c
BaseType_t xTimerReset(TimerHandle_t xTimer, TickType_t xTicksToWait);
```

- 从当前时刻重新开始计时
- 如果定时器未启动，Reset 会启动它
- 适用场景：用户操作超时、看门狗刷新

### `xTimerChangePeriod()` — 改周期

```c
BaseType_t xTimerChangePeriod(TimerHandle_t xTimer,
                              TickType_t xNewPeriod,
                              TickType_t xTicksToWait);
```

- 动态修改定时器周期
- 修改后从当前时刻按新周期计时

## 一次性 vs 自动重载

### 行为对比

```c
/* 一次性定时器：5 秒后执行一次，然后停止 */
TimerHandle_t one_shot = xTimerCreate("oneshot",
    pdMS_TO_TICKS(5000),
    pdFALSE,        // 不重载
    NULL, my_callback);

/* 自动重载定时器：每 1 秒触发一次 */
TimerHandle_t periodic = xTimerCreate("periodic",
    pdMS_TO_TICKS(1000),
    pdTRUE,         // 自动重载
    NULL, my_callback);
```

```
一次性定时器:
   |---5s---|● (触发 → 停止)

自动重载定时器:
   |---1s---|●|---1s---|●|---1s---|●...
```

### 选择指南

| 场景 | 类型 | 理由 |
|------|------|------|
| 超时检测 | 一次性 | 超时后执行一次处理 |
| LED 闪烁 | 自动重载 | 固定频率闪烁 |
| DS18B20 异步读取 | 一次性 | 启动转换后等 750ms 读取 |
| 周期任务调度 | 自动重载 | 定时轮询传感器 |
| 去抖延时 | 一次性 | 按键释放后等 20ms 再采样 |

### 一次性定时器的复用模式

```c
/* 在按键按下时（重新）启动一次性定时器 */
void on_key_press(void)
{
    /* Reset 会重新计时：无论之前还剩多少时间，都从当前开始算 */
    xTimerReset(xDebounceTimer, 0);
}

/* 如果 50ms 内按键没有再按下，执行去抖逻辑 */
void debounce_callback(TimerHandle_t xTimer)
{
    uint8_t stable_state = gpio_get_level(KEY1_GPIO_PIN);
    process_key_state(stable_state);
}
```

## 在项目中的潜在应用

### 应用 1：DS18B20 异步转换

DS18B20 温度转换需要约 750ms。当前方案是 `vTaskDelay(800)` 等待。用软件定时器可以更优雅：

```c
static TimerHandle_t xDs18b20Timer = NULL;

void ds18b20_timer_callback(TimerHandle_t xTimer)
{
    /* 750ms 后转换完成，读取温度 */
    float temp = ds18b20_get_temperature();
    printf("温度: %.2f°C\n", temp);

    /* 更新显示 */
    g_disp.ds18b20_temp = (int16_t)(temp * 10);
    g_disp.dirty = true;
}

void start_temperature_reading(void)
{
    ds18b20_start();                                           // 启动转换
    xTimerStart(xDs18b20Timer, 0);                             // 750ms 后读取
}

void app_main(void)
{
    /* 创建一次性定时器 */
    xDs18b20Timer = xTimerCreate("ds18b20",
                                 pdMS_TO_TICKS(750),
                                 pdFALSE,        // 一次性
                                 NULL,
                                 ds18b20_timer_callback);
}
```

### 应用 2：按键长按检测替代方案

当前按键长按在扫描函数中用计数器实现。用定时器更直观：

```c
static TimerHandle_t xLongPressTimer = NULL;
static uint8_t       xLongPressKeyId = 0;

void long_press_callback(TimerHandle_t xTimer)
{
    uint8_t key_id = xLongPressKeyId;
    printf("K%d 长按触发!\n", key_id + 1);

    /* 执行长按动作 */
    switch (key_id) {
    case 0: rgb_set_color(255, 0, 0); break;
    case 1: rgb_set_color(0, 255, 0); break;
    case 2: rgb_set_color(0, 0, 255); break;
    case 3: rgb_clear(); break;
    }
}

void on_key_press_detected(uint8_t key_id)
{
    xLongPressKeyId = key_id;
    xTimerStart(xLongPressTimer, 0);  // 启动 1 秒定时器
}

void on_key_release_detected(uint8_t key_id)
{
    xTimerStop(xLongPressTimer, 0);   // 释放时取消
}
```

### 应用 3：周期性刷新

```c
static TimerHandle_t xDisplayRefreshTimer = NULL;

void refresh_callback(TimerHandle_t xTimer)
{
    /* 标记脏，让 display_task 在下一帧刷新 */
    g_disp.dirty = true;
}

void app_main(void)
{
    /* 20Hz 刷新定时器 */
    xDisplayRefreshTimer = xTimerCreate("refresh",
                                        pdMS_TO_TICKS(50),
                                        pdTRUE,     // 自动重载
                                        NULL,
                                        refresh_callback);
    xTimerStart(xDisplayRefreshTimer, 0);
}
```

## 重要约束

### 回调在 `tmr Svc` 上下文中运行

**这条规则最重要：** 定时器回调在 `tmr Svc` 任务中运行，**不是你的任务**。这意味着回调的优先级取决于 `tmr Svc` 的优先级，而不是创建定时器的任务。

```
你的任务 (优先级 9)                tmr Svc (优先级 1)
  │                                   │
  ├── xTimerStart() ────────►         │
  │                                   ├── [定时到达] 回调开始
  │                                   │   └── printf / 赋值...  ← 跑在优先级 1！
  │  (正在干其他活)                    │
  │                                   
  回调的优先级 = tmr Svc 的优先级 = 1（不是 9！）
```

**后果：** 你创建定时器的任务优先级再高，回调也只是在优先级 1 上运行。任何优先级 > 1 的任务都能抢占它：

```c
// tmr Svc 优先级为 1 时：
//   - sensor_task (优先级 10) 随时可以抢占回调
//   - key_task    (优先级 9)  随时可以抢占回调  
//   - 如果回调执行时间长，其他定时器都会被推迟

```c
/* ❌ 禁止：在回调中阻塞 tmr Svc 任务 */
void callback(TimerHandle_t xTimer)
{
    vTaskDelay(pdMS_TO_TICKS(10));              // 阻塞 tmr Svc！
    xQueueReceive(queue, &msg, portMAX_DELAY);  // 阻塞 tmr Svc！
    xSemaphoreTake(mutex, portMAX_DELAY);       // 阻塞 tmr Svc！
}

/* ✅ 正确：回调中只做轻量工作 */
void callback(TimerHandle_t xTimer)
{
    g_disp.dirty = true;                        // 快速赋值（微秒级）

    /* 或用非阻塞方式发队列 */
    disp_msg_t msg = { .type = DISP_MSG_TIMEOUT };
    xQueueSend(disp_queue, &msg, 0);            // 不阻塞
}
```

| 回调行为 | 是否允许 | 后果 |
|---------|---------|------|
| `vTaskDelay` / `vTaskDelayUntil` | ❌ 禁止 | 阻塞 tmr Svc，所有定时器推迟 |
| 阻塞式 IPC（Take/Receive 不超时） | ❌ 禁止 | 阻塞 tmr Svc，所有定时器暂停 |
| 非阻塞 IPC（Take/Receive 超时=0） | ✅ 允许 | 不阻塞，快速返回 |
| 全局变量读写 | ✅ 允许 | 微秒级，安全 |
| `printf` / `DBG_INFO` | ⚠️ 小心 | 如果 printf 通过 UART 阻塞，会卡住 tmr Svc |

### 定时器命令队列

所有 `xTimerStart/Stop/Reset/ChangePeriod` 请求通过定时器命令队列发送给 `tmr Svc`。

```
命令队列 ──► [start] [stop] [reset] [change] ──► tmr Svc
```

如果队列满：

```c
if (xTimerStart(xTimer, pdMS_TO_TICKS(100)) != pdPASS) {
    printf("定时器命令队列满，启动失败\n");
}
```

默认定时器命令队列长度（sdkconfig）：
```
CONFIG_FREERTOS_TIMER_QUEUE_LENGTH = 10
```

### ISR 中控制定时器

```c
void IRAM_ATTR isr_handler(void *arg)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    xTimerStartFromISR(xTimer, &xHigherPriorityTaskWoken);
    // xTimerStopFromISR / xTimerResetFromISR 同理

    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

## 常见陷阱

### 1. 定时器未启动

```c
TimerHandle_t xTimer = xTimerCreate("my", 100, pdTRUE, NULL, cb);
// 忘记调用 xTimerStart() → 定时器永远不会触发
xTimerStart(xTimer, 0);  // 记住要 start！
```

### 2. 在回调中操作同一个定时器

```c
void callback(TimerHandle_t xTimer)
{
    /* OK：停止自己 */
    xTimerStop(xTimer, 0);

    /* OK：给自己改周期 */
    xTimerChangePeriod(xTimer, pdMS_TO_TICKS(2000), 0);

    /* OK：复位自己（重新计时） */
    xTimerReset(xTimer, 0);

    /* NOT OK：删除自己（可能导致 use-after-free） */
    // xTimerDelete(xTimer, 0);  // 别这么做
}
```

### 3. 优先级反转（相对于其他任务）

`tmr Svc` 的优先级通常为 `configTIMER_TASK_PRIORITY`（默认 1）。如果定时器回调需要通过队列通知另一个任务，而接收任务的优先级更高，则接收任务会被调度执行，`tmr Svc` 被抢占——但其他定时器的触发也会被推迟。

**解决办法：** 提高 `tmr Svc` 优先级（在 sdkconfig 中修改 `configTIMER_TASK_PRIORITY`），或者让回调只做最轻量的工作。

### 4. 周期小于一个 tick

```c
/* ❌ 错误：10ms tick 下，1ms 周期不可能实现 */
xTimerCreate("fast", pdMS_TO_TICKS(1), pdTRUE, NULL, cb);
// pdMS_TO_TICKS(1) = 0 → 定时器永不触发

/* 需要 <10ms 周期 => 用硬件定时器或 esp_timer */
```

## 总结

| 要点 | 说明 |
|------|------|
| **服务任务** | 所有定时器回调在 `tmr Svc` 任务中运行 |
| **禁止阻塞** | 回调中不能 `vTaskDelay`、不能阻塞式 IPC |
| **两种类型** | `pdTRUE`=自动重载，`pdFALSE`=一次性 |
| **创建后需 Start** | 创建后不自动运行，需调用 `xTimerStart` |
| **ISR 安全** | 有 `FromISR` 版本 |
| **最小周期** | 受 tick 周期限制（100Hz→10ms） |
| **命令队列** | Start/Stop/Reset 通过队列发送，可能满 |
