# FreeRTOS 队列使用指南

## 什么是队列

队列是 FreeRTOS 中**任务间 / ISR→任务**传递数据的主要方式。它是内核对象，天生**线程安全**，通过**数据拷贝**传递。

```
                    ┌──────────────────────┐
                    │       队列            │
 生产者 ──send──►   │ ┌──┐──┐──┐──┐──┐   │  ──receive──►  消费者
                    │ │  │  │  │  │  │   │
                    │ └──┘──┘──┘──┘──┘   │
                    │  length = 5          │
                    └──────────────────────┘
```

### 与信号量的对比

| 同步方式 | 传递什么 | 典型场景 |
|---------|---------|---------|
| **队列** | 数据（任意结构体） | ISR 向任务传递传感器数据 |
| **二值信号量** | "发生了"（无数据） | 简单唤醒，一次一事件 |
| **互斥锁** | 访问权 | 保护共享资源 |
| **事件组** | 多个事件的组合状态 | 一个任务等多个条件同时满足 |

## API 总览

```c
#include "freertos/FreeRTOS.h"
#include "freertos/queue.h"

/* 创建与删除 */
QueueHandle_t xQueueCreate(UBaseType_t uxQueueLength, UBaseType_t uxItemSize);
void          vQueueDelete(QueueHandle_t xQueue);

/* 发送（任务上下文） */
BaseType_t xQueueSend(QueueHandle_t xQueue, const void *pvItemToQueue, TickType_t xTicksToWait);
BaseType_t xQueueSendToFront(QueueHandle_t xQueue, const void *pvItemToQueue, TickType_t xTicksToWait);
BaseType_t xQueueSendToBack(QueueHandle_t xQueue, const void *pvItemToQueue, TickType_t xTicksToWait);

/* 发送（ISR 上下文） */
BaseType_t xQueueSendFromISR(QueueHandle_t xQueue, const void *pvItemToQueue, BaseType_t *pxHigherPriorityTaskWoken);
BaseType_t xQueueSendToBackFromISR(QueueHandle_t xQueue, const void *pvItemToQueue, BaseType_t *pxHigherPriorityTaskWoken);

/* 接收 */
BaseType_t xQueueReceive(QueueHandle_t xQueue, void *pvBuffer, TickType_t xTicksToWait);
BaseType_t xQueuePeek(QueueHandle_t xQueue, void *pvBuffer, TickType_t xTicksToWait);  // 不删除数据

/* 查询 */
UBaseType_t uxQueueMessagesWaiting(QueueHandle_t xQueue);      // 当前消息数
UBaseType_t uxQueueSpacesAvailable(QueueHandle_t xQueue);      // 剩余空间
BaseType_t  xQueueIsQueueEmptyFromISR(QueueHandle_t xQueue);
BaseType_t  xQueueIsQueueFullFromISR(QueueHandle_t xQueue);
```

## 队列创建

### `xQueueCreate()`

```c
QueueHandle_t xQueueCreate(
    UBaseType_t uxQueueLength,  // 队列可容纳的元素数量
    UBaseType_t uxItemSize      // 每个元素的大小（bytes）
);
```

- **返回：** 队列句柄，内存不足返回 NULL
- 内存分配在 FreeRTOS 堆中，大小 = `uxQueueLength × (uxItemSize + 控制块开销)`

**示例：**

```c
/* 传递传感器数据 */
typedef struct {
    int16_t temperature;
    uint8_t humidity;
} sensor_data_t;

QueueHandle_t xSensorQueue = xQueueCreate(5, sizeof(sensor_data_t));
if (xSensorQueue == NULL) {
    printf("队列创建失败！\n");
}
```

### 队列容量选择

| 场景 | 建议长度 | 理由 |
|------|---------|------|
| ISR → 任务（高频） | 5~10 | 防止 ISR 频繁丢弃数据 |
| 任务 → 任务（周期） | 1~3 | 一次一帧，最新数据覆盖旧数据 |
| 命令/控制 | 5~10 | 缓冲多条未处理的命令 |

## 发送与接收

### `xQueueSend()` — 任务中发送

```c
BaseType_t xQueueSend(
    QueueHandle_t xQueue,         // 队列句柄
    const void *  pvItemToQueue,  // 指向要发送的数据
    TickType_t    xTicksToWait    // 队列满时的等待时间
);
```

- 数据被**拷贝**到队列内部缓冲区（不是存指针）
- `xQueueSendToBack` = `xQueueSend`（追加到尾部）
- `xQueueSendToFront` = 插入到头部（紧急消息）

**阻塞行为：**

```
调用 xQueueSend()
    │
    ├── 队列有空位 ──► 拷贝数据到队列，返回 pdPASS
    │
    └── 队列无空位 ──► 阻塞 xTicksToWait
            │
            ├── 期间有接收 ──► 写入，返回 pdPASS
            │
            └── 超时 ──► 返回 errQUEUE_FULL
```

### `xQueueReceive()` — 任务中接收

```c
BaseType_t xQueueReceive(
    QueueHandle_t xQueue,        // 队列句柄
    void *        pvBuffer,      // 接收缓冲区指针
    TickType_t    xTicksToWait   // 队列空时的等待时间
);
```

- 数据从队列**移除**（`xQueuePeek` 不移除）
- 缓冲区大小**必须 ≥** 创建时的 `uxItemSize`，否则会内存越界

**阻塞模式：**

```c
/* 模式一：永久阻塞 — 等到有数据 */
xQueueReceive(queue, &buf, portMAX_DELAY);

/* 模式二：超时等待 — 100ms 内没数据就放弃 */
if (xQueueReceive(queue, &buf, pdMS_TO_TICKS(100)) == pdPASS) {
    process(&buf);
} else {
    timeout_handler();
}

/* 模式三：非阻塞轮询 — 有就处理，没有就干别的 */
if (xQueueReceive(queue, &buf, 0) == pdPASS) {
    process(&buf);
}
```

### `xQueueSendFromISR()` — ISR→任务

```c
BaseType_t xQueueSendFromISR(
    QueueHandle_t xQueue,
    const void *  pvItemToQueue,
    BaseType_t *  pxHigherPriorityTaskWoken  // 出参
);
```

ISR 中的标准用法：

```c
void IRAM_ATTR sensor_isr_handler(void *arg)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    sensor_data_t data = read_sensor();

    xQueueSendFromISR(sensor_queue, &data, &xHigherPriorityTaskWoken);

    /* 如果接收此队列的任务优先级高于当前任务，立即切换 */
    if (xHigherPriorityTaskWoken) {
        portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
    }
}
```

> `xQueueSendFromISR` 的 `pxHigherPriorityTaskWoken` 传 NULL 不会 crash，但上下文切换会延迟到下一个 tick 中断。

## 在项目中的使用实例

### 实例：OLED 显示消息队列

本项目使用 `disp_queue` + 脏标志双驱动模式：

```c
/* oled_display.c — 队列创建 */
disp_queue = xQueueCreate(5, sizeof(disp_msg_t));

/* disp_msg_t 定义 */
typedef struct {
    uint8_t  type;               // DISP_MSG_KEY_PRESS, DISP_MSG_SENSOR_UPDATE...
    int16_t  val1;               // 数据字段 1
    int16_t  val2;               // 数据字段 2
    int16_t  val3;               // 数据字段 3
} disp_msg_t;
```

**显示任务 — 队列接收 + 脏标志合并：**

```c
void display_task(void *pv)
{
    disp_msg_t msg;
    while (1) {
        /* 1. 非阻塞收完所有消息（同一 tick 内的多条更新） */
        while (xQueueReceive(disp_queue, &msg, 0) == pdPASS) {
            switch (msg.type) {
            case DISP_MSG_SENSOR_UPDATE:
                g_disp.ds18b20_temp = msg.val1;
                g_disp.dht11_temp   = msg.val2;
                g_disp.dht11_humi   = msg.val3;
                g_disp.dirty = true;          // 标记脏
                break;
            case DISP_MSG_KEY_PRESS:
                g_disp.key_press = msg.val1;
                g_disp.dirty = true;
                break;
            }
        }

        /* 2. 脏时统一渲染（一次 I2C 传输，不是每条消息刷一次） */
        if (g_disp.dirty) {
            oled_clear_gram();
            draw_all_content();
            oled_refresh_gram();
            g_disp.dirty = false;
        }

        vTaskDelay(pdMS_TO_TICKS(20));   // 50Hz 刷新
    }
}
```

**按键任务 — 发送消息：**

```c
/* key.c — K1 按下 */
case KEY_IND_EVENT_PRESS:
    disp_msg_t msg = { .type = DISP_MSG_KEY_PRESS, .val1 = 1 };
    xQueueSend(disp_queue, &msg, 0);
    break;
```

### 实例：传感器数据采集

```c
/* sensor_task 发送采集结果 */
void sensor_task(void *pv)
{
    while (1) {
        ds18b20_start();
        vTaskDelay(pdMS_TO_TICKS(800));     // 等待转换完成

        disp_msg_t msg = {
            .type = DISP_MSG_SENSOR_UPDATE,
            .val1 = ds18b20_get_temperature(),
        };
        dht11_read_data((uint8_t *)&msg.val2, (uint8_t *)&msg.val3);

        /* 发送到显示队列 */
        xQueueSend(disp_queue, &msg, pdMS_TO_TICKS(10));

        vTaskDelay(pdMS_TO_TICKS(200));
    }
}
```

### 实例：ISR 传递按键事件

未来切换到中断驱动模式时：


## 队列阻塞机制详解

### 多任务等待同一个队列

```
任务 A (优先级 7)  ──► 阻塞在 Queue_A（等待接收）
任务 B (优先级 5)  ──► 阻塞在 Queue_A（等待接收）
                           │
        某任务发送 ─────────┤
                           │
                    ┌──────┴──────┐
                    │ 谁被唤醒？   │
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │ 优先级最高的  │
                    │ 任务 A 被唤醒 │
                    └─────────────┘
```

如果多个任务阻塞在同一个队列上等待接收，数据到达时**优先级最高**的任务被唤醒。

### 队列满时发送者阻塞

```
队列长度 = 3，当前有 3 条消息
        │
发送者 D 调用 xQueueSend(..., pdMS_TO_TICKS(100))
        │
        ├── 队列满，D 阻塞
        │
        ├── 接收者取走一条 ──► D 被唤醒，写入数据
        │
        └── 100ms 超时 ──► D 仍然阻塞，返回 errQUEUE_FULL
```

## 使用注意事项

### 1. ISR 中使用队列

```c
/* ❌ 错误：ISR 中不能用非 FromISR 版本 */
void IRAM_ATTR isr(void *arg) {
    xQueueSend(queue, &data, 0);  // ISR 中非法
}

/* ✅ 正确 */
void IRAM_ATTR isr(void *arg) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(queue, &data, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

### 2. 队列满的处理

ISR 中队列满时，`xQueueSendFromISR` **不会阻塞**，而是直接返回 `errQUEUE_FULL`。数据会丢失，所以队列长度要足够。

```c
/* 建议：ISR 发送的队列，长度设 5~10 缓冲峰值 */
#define ISR_QUEUE_LENGTH 10
```

### 3. 传递指针的风险

```c
/* ❌ 危险：传指针，但指针指向的数据可能在接收时已失效 */
char buffer[64];
snprintf(buffer, 64, "temp: %d", temp);
xQueueSend(queue, &buffer, 0);  // 只拷贝了指针？不对，队列会拷贝 buffer 的内容

/* ✅ 正确用法：传结构体值拷贝 */
typedef struct {
    char data[64];
    uint8_t len;
} msg_t;
msg_t msg;
snprintf(msg.data, 64, "temp: %d", temp);
msg.len = strlen(msg.data);
xQueueSend(queue, &msg, 0);  // 拷贝整个结构体到队列
```

### 4. 队列 vs 全局变量

| 方式 | 线程安全 | 阻塞 | 数据拷贝 | 适用场景 |
|------|---------|------|---------|---------|
| **队列** | ✅ 内核保证 | ✅ 支持 | 自动拷贝 | ISR→任务，任务间数据流 |
| **全局变量** | ❌ 需要互斥锁 | ❌ | 不拷贝 | 低频少量状态共享 |
| **全局变量 + dirty** | ⚠️ 单核安全 | ❌ | 不拷贝 | 本项目的显示数据 |

---

## 总结

| 要点 | 说明 |
|------|------|
| **线程安全** | 队列本身是线程安全的，无需额外加锁 |
| **数据拷贝** | 发送方数据被拷贝到队列内部缓冲区 |
| **ISR 安全版** | 用 `xQueueSendFromISR` + `portYIELD_FROM_ISR` |
| **阻塞机制** | 队列满时发送者阻塞，队列空时接收者阻塞 |
| **超时 0** = 不阻塞 | 适合轮询模式 |
| **队列长度** | ISR→任务的队列应设足够长度（5~10）避免丢数据 |
| **数据生命周期** | 队列拷贝后发送方的数据可立即释放 |
