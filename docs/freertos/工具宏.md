# FreeRTOS 工具宏使用指南

## 概述

FreeRTOS 提供了一系列工具宏，用于 **时间转换**、**优先级管理**、**ISR 上下文切换** 等常用操作。这些宏在整个 FreeRTOS API 中频繁出现，理解它们的行为对避免 bug 至关重要。

```c
#include "freertos/FreeRTOS.h"
```

---

## `pdMS_TO_TICKS()` — 毫秒转 Tick

最常用的宏。将毫秒转换为调度器的 tick 数。

```c
TickType_t pdMS_TO_TICKS(uint32_t xTimeInMs);
```

### 基本用法

```c
vTaskDelay(pdMS_TO_TICKS(100));                 // 延时 100ms
xQueueReceive(queue, &buf, pdMS_TO_TICKS(50));  // 等 50ms
xSemaphoreTake(sem, pdMS_TO_TICKS(1000));       // 等 1 秒
```

### ⚠️ 关键陷阱：截断为 0

在默认 **100Hz**（每 tick = 10ms）配置下：

| 输入 ms | 计算结果 | 实际效果 |
|---------|---------|---------|
| `pdMS_TO_TICKS(1000)` | 100 ticks | ✅ 等 1 秒 |
| `pdMS_TO_TICKS(100)` | 10 ticks | ✅ 等 100ms |
| `pdMS_TO_TICKS(10)` | 1 tick | ✅ 等 10ms |
| `pdMS_TO_TICKS(9)` | **0** tick | ❌ 不延时 |
| `pdMS_TO_TICKS(0)` | **0** tick | = `vTaskDelay(0)` |

```c
/* ❌ bug：本想等 9ms，实际等了 0ms */
vTaskDelay(pdMS_TO_TICKS(9));

/* ✅ 安全：至少 10ms */
vTaskDelay(pdMS_TO_TICKS(10));
```

### 补救方案

如果确实需要 <10ms 的延时：

```c
/* 方案 1：用 esp_rom_delay_us（旋转等待，不交出 CPU） */
esp_rom_delay_us(9000);   // 等 9ms（阻塞当前核，不用于任务）

/* 方案 2：提高 tick 频率（全局影响） */
// menuconfig → FreeRTOS → Tick rate (Hz) → 1000
// 代价：CPU 每秒处理 1000 次 tick 中断（原 100 次），开销增加

/* 方案 3：用硬件定时器或 esp_timer（推荐） */
#include "esp_timer.h"
esp_timer_create_args_t args = { .callback = &my_callback };
esp_timer_handle_t timer;
esp_timer_create(&args, &timer);
esp_timer_start_once(timer, 9000);  // 9ms
```

### 编译时检查

```c
/* 如果 tick 率太低，在编译时就能发现 */
#if pdMS_TO_TICKS(10) == 0
#error "Tick rate too low! Need at least 100Hz for 10ms resolution."
#endif
```

### 计算原理

```
pdMS_TO_TICKS(ms) = (ms + portTICK_PERIOD_MS - 1) / portTICK_PERIOD_MS
                  = (ms + 9) / 10   // 在 100Hz 下

portTICK_PERIOD_MS = 1000 / configTICK_RATE_HZ
```

注：实际实现是向上取整整数除法，但小于一个 tick 的值结果为 0。

## `portYIELD_FROM_ISR()` — ISR 上下文切换

### 作用

ISR 末尾调用，如果条件满足则触发上下文切换，让被唤醒的高优先级任务**立即**运行。

```c
void portYIELD_FROM_ISR(BaseType_t xHigherPriorityTaskWoken);
```

### 标准用法

```c
void IRAM_ATTR some_isr(void *arg)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    /* 调用 FromISR API，它们可能设置 xHigherPriorityTaskWoken = pdTRUE */
    xQueueSendFromISR(queue, &data, &xHigherPriorityTaskWoken);
    xEventGroupSetBitsFromISR(eg, BIT0, &xHigherPriorityTaskWoken);
    vTaskNotifyGiveFromISR(task, &xHigherPriorityTaskWoken);

    /* 统一在末尾检查是否需要立刻切换 */
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

### 工作原理

```
ISR 触发
  │
  ├── xQueueSendFromISR() → 唤醒一个高优先级任务
  │     └── xHigherPriorityTaskWoken = pdTRUE
  │
  └── portYIELD_FROM_ISR(pdTRUE)
        └── 中断返回时 → 不回到被中断的任务
              └── 直接运行被唤醒的高优先级任务
```

### 传 NULL 的后果

```c
/* 可以传 NULL，但上下文切换会延迟到下一个 tick */
xQueueSendFromISR(queue, &data, NULL);
// 不立即切换，被唤醒的任务等最多 10ms 才能运行
```

```c
/* 完整写法，不偷懒 */
BaseType_t xHigherPriorityTaskWoken = pdFALSE;
xQueueSendFromISR(queue, &data, &xHigherPriorityTaskWoken);
portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
```

### `FromISR` 命名规则

FreeRTOS 中所有带 `FromISR` 后缀的函数都是 ISR 安全版本：

| ISR 安全版本 | 非 ISR 版本 |
|-------------|------------|
| `xQueueSendFromISR` | `xQueueSend` |
| `xSemaphoreGiveFromISR` | `xSemaphoreGive` |
| `xEventGroupSetBitsFromISR` | `xEventGroupSetBits` |
| `vTaskNotifyGiveFromISR` | `xTaskNotifyGive` |
| `xTimerStartFromISR` | `xTimerStart` |

## `portMAX_DELAY` — 永久等待

### 作用

```c
const TickType_t portMAX_DELAY;
```

表示「永久等待」——任务将阻塞直到条件满足。

### 使用示例

```c
xSemaphoreTake(mutex, portMAX_DELAY);               // 等到拿到锁
xQueueReceive(queue, &buf, portMAX_DELAY);           // 等到有数据
xEventGroupWaitBits(eg, BIT0, pdTRUE, pdTRUE,        // 等到 bit0 置位
                    portMAX_DELAY);
ulTaskNotifyTake(pdTRUE, portMAX_DELAY);             // 等到有通知
```

### 注意事项

| 条件 | 行为 |
|------|------|
| 调度器运行中 | 永久阻塞，直到条件满足 |
| 调度器未启动 | `portMAX_DELAY` 立即返回 0（条件不满足） |
| 在 ISR 中使用 | **禁止**，ISR 不能阻塞 |
| 与 TWDT 配合 | 长时间 `portMAX_DELAY` 阻塞会自动喂狗（任务不在运行态） |

### 超时与永久的判断

```c
/* pdMS_TO_TICKS 定义超时 */
xQueueReceive(queue, &buf, pdMS_TO_TICKS(100));   // 最多等 100ms

/* portMAX_DELAY 永久等待 */
xQueueReceive(queue, &buf, portMAX_DELAY);        // 等一辈子

/* 0 = 不阻塞 */
xQueueReceive(queue, &buf, 0);                    // 轮询，立即返回
```

## `portTICK_PERIOD_MS` — Tick 周期

```c
const TickType_t portTICK_PERIOD_MS;
```

每个 tick 的毫秒数：

| Tick 频率 | `portTICK_PERIOD_MS` |
|-----------|-------------------|
| 100 Hz（默认） | 10 |
| 1000 Hz | 1 |

### 用途

```c
/* 计算期望的 tick 数 */
TickType_t ticks = 100 / portTICK_PERIOD_MS;   // 100ms 需要多少 tick

/* 打印当前 tick 周期 */
printf("Tick 周期: %lu ms\n", portTICK_PERIOD_MS);
```

> `pdMS_TO_TICKS(x)` 内部就是通过 `portTICK_PERIOD_MS` 计算的，所以一般不需要直接使用它。

## 优先级相关常量

### `configMAX_PRIORITIES`

ESP-IDF 默认值：**25**

```c
#define configMAX_PRIORITIES 25
```

- 优先级范围：**0 ~ 24**
- **数字越大，优先级越高**
- **0 保留**给空闲任务（IDLE）

```c
/* 本项目的优先级分配 */
#define PRIO_IDLE     0   // 空闲任务
#define PRIO_SENSOR  10   // 传感器采集
#define PRIO_KEY      9   // 按键扫描
#define PRIO_DISPLAY  8   // OLED 显示
```

### `tskIDLE_PRIORITY`

```c
#define tskIDLE_PRIORITY 0
```

空闲任务的优先级。任何优先级 > 0 的任务都会抢占空闲任务。

### `configTIMER_TASK_PRIORITY`

定时器服务任务（`tmr Svc`）的优先级。ESP-IDF 默认值：**1**。

```c
#define configTIMER_TASK_PRIORITY 1
```

如果定时器回调需要被及时响应，可以调高这个值。

## 任务创建相关宏

### `tskNO_AFFINITY`

```c
#define tskNO_AFFINITY 0x7FFF
```

在 `xTaskCreatePinnedToCore` 中表示不绑定特定核心，由调度器自由调度：

```c
xTaskCreatePinnedToCore(task, "name", 2048, NULL, 5, NULL,
                        tskNO_AFFINITY);   // 不绑定核心
```

### `configMINIMAL_STACK_SIZE`

```c
#define configMINIMAL_STACK_SIZE 768
```

空闲任务的最小栈大小。**ESP-IDF 中单位是 bytes**。

> **不要直接用此值创建用户任务。** ESP-IDF 建议用户任务栈至少 2048 bytes。`configMINIMAL_STACK_SIZE` 仅保证空闲任务和 `tmr Svc` 能正常运行。

## 常用返回值宏

```c
#define pdTRUE  1
#define pdFALSE 0
#define pdPASS  1
#define pdFAIL  0
```

```c
/* 检查 API 返回值 */
if (xQueueSend(queue, &data, 0) == pdPASS) {   // 或 == pdTRUE
    printf("发送成功\n");
} else {
    printf("队列满\n");
}

if (xSemaphoreTake(mutex, pdMS_TO_TICKS(100)) != pdPASS) {
    printf("获取互斥锁超时\n");
}
```

### 错误码

| 宏 | 值 | API | 含义 |
|---|-----|-----|------|
| `errQUEUE_FULL` | 0 | `xQueueSend` | 队列满 |
| `errQUEUE_EMPTY` | 0 | `xQueueReceive` | 队列空 |
| `errCOULD_NOT_ALLOCATE_REQUIRED_MEMORY` | -1 | `xTaskCreate` | 内存不足 |

> `errQUEUE_FULL` 和 `errQUEUE_EMPTY` 都等于 0（`pdFALSE`）。`xQueueSend` 返回 0 既可能是队列满，也可能是销毁了。一般用 `pdPASS` / `pdFALSE` 判断即可。

## 配置相关的宏

以下宏在 `sdkconfig` 或 `FreeRTOSConfig.h` 中配置：

| 宏 | 默认值 | 含义 | 修改方式 |
|---|-------|------|---------|
| `configTICK_RATE_HZ` | 100 | Tick 频率（Hz） | `idf.py menuconfig` |
| `configMAX_PRIORITIES` | 25 | 最大优先级数 | `idf.py menuconfig` |
| `configMINIMAL_STACK_SIZE` | 768 | 最小栈大小（bytes） | `idf.py menuconfig` |
| `configTIMER_TASK_PRIORITY` | 1 | 定时器服务任务优先级 | `idf.py menuconfig` |
| `configTIMER_QUEUE_LENGTH` | 10 | 定时器命令队列长度 | `idf.py menuconfig` |
| `configUSE_TRACE_FACILITY` | 0 | 启用跟踪功能 | `idf.py menuconfig` |
| `configUSE_STATS_FORMATTING_FUNCTIONS` | 0 | 启用统计格式化 | `idf.py menuconfig` |

```c
/* 在代码中读取这些值 */
printf("Tick 频率: %d Hz\n", configTICK_RATE_HZ);
printf("最大优先级: %d\n", configMAX_PRIORITIES);
printf("最小栈大小: %d bytes\n", configMINIMAL_STACK_SIZE);
```

## 常见用法速查

```c
/* ---------- 延时 ---------- */
vTaskDelay(pdMS_TO_TICKS(100));                 // 延时 100ms

/* ---------- 队列操作 ---------- */
xQueueSend(queue, &data, pdMS_TO_TICKS(50));    // 最多等 50ms
xQueueReceive(queue, &buf, portMAX_DELAY);      // 永久等待

/* ---------- 信号量 ---------- */
xSemaphoreTake(sem, pdMS_TO_TICKS(1000));       // 等 1 秒
xSemaphoreGive(sem);                             // 立即释放

/* ---------- ISR ---------- */
BaseType_t xHigherPriorityTaskWoken = pdFALSE;
xQueueSendFromISR(queue, &data, &xHigherPriorityTaskWoken);
portYIELD_FROM_ISR(xHigherPriorityTaskWoken);

/* ---------- 判断返回值 ---------- */
if (xQueueSend(queue, &data, 0) != pdPASS) {
    // 队列满
}
```

## 总结

| 宏 / 常量 | 用途 | 关键陷阱 |
|-----------|------|---------|
| `pdMS_TO_TICKS(x)` | ms 转 tick | `x < 10` 结果为 0 |
| `portYIELD_FROM_ISR(x)` | ISR 中触发调度 | 必须传 `FromISR` API 的出参 |
| `portMAX_DELAY` | 永久阻塞 | 不能在 ISR 中用 |
| `portTICK_PERIOD_MS` | 每个 tick 的 ms 数 | 100Hz → 10ms |
| `configMAX_PRIORITIES` | 最大优先级数 | 默认 25（0~24） |
| `tskNO_AFFINITY` | 不绑定核心 | 仅 ESP-IDF 有 |
| `configMINIMAL_STACK_SIZE` | 最小栈 | 单位 bytes |
