# DHT11 温湿度传感器协议

## 概述

DHT11 是广州奥松（AoSong）生产的数字温湿度传感器，使用单总线（Single-Bus）协议与主机通信。量程：温度 0~50°C（±2°C），湿度 5~95%RH（±5%RH），采样周期 ≥1 秒。

## 物理层

- 一根数据线 DQ：双向数据输入/输出（漏极开路 + 外部上拉电阻）
- GPIO 配置为 **开漏模式**（`GPIO_MODE_INPUT_OUTPUT_OD`）
- 空闲状态：高电平（由上拉电阻维持）
- DHT11 主动拉低总线表示应答，主动输出高低电平表示数据 0/1

## 通信时序

DHT11 的启动信号需要 **毫秒级** 拉低（`vTaskDelay()`），位级时序使用 `esp_rom_delay_us()` 微秒级延时，**没有独立的时钟线**。

### 主机启动信号

```
主机:   ┌────────────────────────────┐   ┌──┐
        │  拉低 ≥18ms（驱动给 25ms）  │   │  │  释放 + 30μs
        └────────────────────────────┘   └──┘
```
主机拉低 ≥18ms → 释放 → 拉高 20~40μs

### DHT11 应答

```
DHT11:                      ┌──────────────────┐
                            │  拉低 40~80μs    │   ┌──────
                            └──────────────────┘   │
                                                   │ 拉高 40~80μs
                                                   └──────
```
DHT11 拉低 40~80μs 作为应答 → 拉高 40~80μs 准备数据

### 数据位时序

DHT11 连续发送 40 位数据，每位由 DHT11 控制时序：

```
每一位的开始:    DHT11 拉低 50μs
                ┌────┐
                │    │               ┌───────────
                └────┘───────────────┤
                 前导低电平 50μs      └── 高电平宽度决定 0/1

    逻辑 "0":     高电平 26~28μs
                ┌────┐    ┌──┐
                │    │    │  │   ← 数据采样点（主机延时 40μs 后采样）
                └────┘    └──┘

    逻辑 "1":     高电平 70μs
                ┌────┐    ┌──────────┐
                │    │    │          │   ← 数据采样点
                └────┘    └──────────┘
```

| 数据位 | 低电平时间 | 高电平时间 |
|--------|-----------|-----------|
| 0 | 50μs | 26~28μs |
| 1 | 50μs | 70μs |

### 数据传输

- **高位优先**：读写字节时先传 MSB（bit7），后传 LSB（bit0），与 DS18B20 相反
- 每个位约 120μs，一个字节约 1ms

## 温湿度读取流程

```
┌─────────────────────┐
│  dht11_reset()      │ 主机拉低 ≥18ms + 释放 30μs
│  启动信号            │ DHT11 在此期间完成温湿度采样
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  dht11_check()      │ 等待 DHT11 应答
│  检测 DHT11 是否存在 │ 拉低 80μs + 拉高 80μs
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  dht11_read_byte()  │ Byte0 — 湿度整数
│  dht11_read_byte()  │ Byte1 — 湿度小数（DHT11 恒为 0）
│  dht11_read_byte()  │ Byte2 — 温度整数
│  dht11_read_byte()  │ Byte3 — 温度小数（DHT11 恒为 0）
│  dht11_read_byte()  │ Byte4 — 校验和
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  校验: buf[4] ==    │
│  buf[0]+...+buf[3]  │ 校验通过 → 输出温湿度
│                     │ 校验失败 → 丢弃数据
└─────────────────────┘
```

## 数据格式

### 40 位数据包

5 字节，高位优先（MSB first）：

```
 Byte 0      Byte 1      Byte 2      Byte 3      Byte 4
┌─────────┬──────────┬──────────┬──────────┬──────────────┐
│ 湿度整数 │ 湿度小数  │ 温度整数  │ 温度小数  │    校验和     │
│   %RH   │ (恒为 0)  │   °C     │ (恒为 0)  │ 前4字节累加和 │
└─────────┴──────────┴──────────┴──────────┴──────────────┘
    bit7→0    bit7→0     bit7→0     bit7→0     bit7→0
       高位优先（MSB first）────→
```

校验和 = (Byte0 + Byte1 + Byte2 + Byte3) & 0xFF

### 示例

```
收到:  00100101  00000000  00011100  00000000  01000001
       ────────  ────────  ────────  ────────  ────────
       0x25=37   0x00=0    0x1C=28   0x00=0    0x41=65

湿度 = 37%RH
温度 = 28°C
校验: 37 + 0 + 28 + 0 = 65  ✓
```

## 驱动 API

| 函数 | 说明 |
|------|------|
| `dht11_init()` | 初始化 GPIO（开漏模式），复位检测，返回 0=正常 |
| `dht11_reset()` | 拉低 25ms 复位总线，释放 30μs |
| `dht11_check()` | 等待应答，返回 0=存在 |
| `dht11_read_bit()` | 读一个位（等待前导低 → 等待高 → 延时 40μs 后采样） |
| `dht11_read_byte()` | 读一个字节（高位优先） |
| `dht11_read_data()` | 读取温湿度，输出整数温度°C 和整数湿度%RH |

## 与 DS18B20 的差异

| 特性 | DHT11 | DS18B20 |
|------|-------|---------|
| 测量内容 | 温度 + 湿度 | 温度 |
| 精度 | ±2°C / ±5%RH | ±0.5°C |
| 分辨率 | 1°C / 1%RH | 0.0625°C（12位） |
| 数据位序 | **高位优先（MSB first）** | 低位优先（LSB first） |
| 启动信号 | 拉低 **≥18ms** | 拉低 750μs |
| 主机主动写数据 | 不支持（仅 DHT11 发数据） | 支持（双向读写） |
| 寻址 | 不支持 | 支持（64位ROM码） |
| 采样间隔 | ≥1 秒 | 无限制 |

## 常见问题

### 1. 为什么 DHT11 启动需要拉低 ≥18ms？

DHT11 内部使用低电平持续时间作为"唤醒 + 开始采样"的信号。此期间 DHT11 完成一次温湿度测量。低于 18ms 可能不被识别或采样不完整。

### 2. 采样间隔至少 1 秒

DHT11 数据手册明确要求两次读取之间至少间隔 1 秒。频率过高会导致传感器内部温度升高，读数不准。参考驱动中主循环每 1 秒调用一次 `dht11_read_data()`。

### 3. DHT11 小数部分为什么是 0？

DHT11 是低精度传感器，小数部分始终为 0。如果需要小数精度，应使用 DHT22（精度 ±0.5°C，含小数）。

### 4. 与 DS18B20 共用引脚

DHT11 和 DS18B20 的参考代码都使用 GPIO13。项目中 DS18B20 已占用 GPIO13，故 DHT11 默认设为 GPIO4。两个传感器都使用开漏模式 + 上拉电阻，可独立接线，**不可直接共线**（通信协议不同，会互相干扰）。

### 5. 线长限制

3.3V 供电时，DHT11 的连线建议不超过 20 米。超过此长度时信号反射和压降可能导致通信失败。
