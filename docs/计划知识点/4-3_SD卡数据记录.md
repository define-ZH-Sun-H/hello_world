# 4-3 SD 卡数据记录

## 1. SD 卡在 ESP32-S3 上的两种接口

ESP32-S3 可以通过两种方式读写 SD 卡，它们共享同一个 FAT 文件系统层，区别只在于物理接口。

| 特性 | SDMMC（SD 专用控制器） | SDSPI（SPI 总线模拟） |
|------|----------------------|---------------------|
| 速度 | 最快 40MB/s（4-bit DDR） | 通常 10-20MB/s |
| 引脚数 | 6 根（CLK + CMD + D0-D3） | 4 根（CLK + MOSI + MISO + CS） |
| 硬件要求 | 专用 SDMMC 控制器，引脚有固定映射 | 任意 SPI 空闲引脚即可 |
| CPU 占用 | 低（硬件控制） | 高（SPI 中断驱动） |
| 热插拔 | 支持 | 支持 |
| 适用场景 | 高速数据记录、拍照存图 | 低频数据记录、引脚受限场景 |

### 本项目用法

普中 ESP32-S3 开发板上的 SD 卡座通过 **SDSPI 模式**驱动（官方例程使用 SPI2 总线映射 SD 信号）。

| 连接 | SD 信号名 | GPIO 引脚 |
|------|----------|-----------|
| MISO | DAT0 | GPIO 40 |
| MOSI | CMD  | GPIO 38 |
| CLK  | CLK  | GPIO 39 |
| CS   | CD/DAT3 | GPIO 1 |

驱动文件在 [components/bsp/sd/](../../components/bsp/sd/) 下：
- `sd.h` — 公开接口声明
- `sd.c` — SPI 总线初始化 + FATFS 挂载 + 容量查询

初始化在 `app_main()` Phase 1 末尾调用，非阻塞——无 SD 卡时自动跳过，不影响启动。

## 2. SD 卡与 SPIFFS 的本质区别

初学者最容易混淆的是 SD 卡和 SPIFFS 都是"存文件"，但它们完全不同：

| | SPIFFS | SD 卡（FAT） |
|--|--------|-------------|
| 存储介质 | **内部 Flash**（焊死在板子上） | **可插拔 SD 卡** |
| 文件系统 | SPIFFS（嵌入式专用） | FAT32 / exFAT（与 PC 兼容） |
| 容量 | 分区大小（本项目 2MB） | 取决于 SD 卡（通常 4-32GB） |
| 拔掉会怎样 | 不可能拔掉 | 可以，但必须先 unmount |
| 目录支持 | ❌ 不支持 | ✅ 支持，和电脑一样 |
| 插电脑能读？ | ❌ 不能 | ✅ 能，插读卡器直接打开 |
| 写入速度 | 慢（Flash 写入特性） | 快（取决于 SD 卡等级） |
| 掉电安全 | ✅ 日志结构，天然掉电安全 | ❌ 写入中掉电可能损坏文件系统 |

**一句话总结：** SPIFFS 存"板子自己的配置"，SD 卡存"要拿出来用的大数据"（日志、照片）。

## 3. FAT 文件系统基础（给初学者的解释）

SD 卡用 FAT32 文件系统——就是你在 Windows 上看到的 U 盘格式。

### 3.1 什么是文件系统

文件系统就是一套规则，解决"数据在 Flash 上到底存在哪个位置"的问题。

没有文件系统时，你要自己记住"温度数据从 0x1000 地址开始存，长度 512 字节"。有了文件系统，你只需要说"写到 `/sdcard/data.csv` 这个文件里"，它自动帮你管理地址分配、空间回收。

### 3.2 FAT32 的关键限制

| 限制 | 值 | 影响 |
|------|----|------|
| 单文件最大 | **4GB** | 日常日志不存在这个问题 |
| 文件名最大长度 | 8.3 格式（短名）或 255 字符（长名） | 够用 |
| 根目录最大文件数 | 取决于扇区大小，通常 512 个 | 日志文件别放在根目录下用子文件夹 |
| 碎片 | FAT 容易碎片化，尤其频繁删写小文件 | 少删写、用追加写 |

### 3.3 SD 卡的分区结构

一张 SD 卡出厂时已经分好区、格式化成 FAT32 了。我们要做的是：

```
SD 卡（出厂状态）
┌───────────────────────────────┐
│ MBR (主引导记录)               │ ← 分区表，描述"第1个分区从哪开始到哪结束"
├───────────────────────────────┤
│ 第1分区: FAT32 文件系统        │ ← 这就是我们挂载后读写的地方
│  ├─ 引导扇区                  │
│  ├─ FAT 表（文件分配表）       │ ← 记录每个文件占用了哪些簇
│  ├─ 根目录区                  │ ← 记录文件/文件夹的名字和属性
│  └─ 数据区                    │ ← 文件内容真正存放的地方
└───────────────────────────────┘
```

ESP-IDF 的 `esp_vfs_fat_sdmmc_mount()` 自动处理了这一切，你不需要手动解析 MBR 或 FAT 表。

## 4. 挂载流程详解

### 4.1 最简单的挂载方式

ESP-IDF 提供了"一站式"挂载函数，把 SDMMC 初始化 + FAT 挂载 + VFS 注册打包成一个调用：

```c
#include "esp_vfs_fat.h"
#include "sdmmc_cmd.h"
#include "driver/sdmmc_host.h"

#define MOUNT_POINT "/sdcard"

void mount_sd_card(void)
{
    // 1. 配置挂载选项
    esp_vfs_fat_mount_config_t mount_config = {
        .format_if_mount_failed = false,  // 挂载失败不格式化（保护现有数据）
        .max_files = 5,                   // 同一时间最多打开 5 个文件
        .allocation_unit_size = 16 * 1024,// 分配单元 16KB（提高大文件性能）
    };

    sdmmc_card_t *card;                    // SD 卡信息（后续操作需要）

    // 2. 获取 SDMMC 主机默认配置
    sdmmc_host_t host = SDMMC_HOST_DEFAULT();

    // 3. 配置 SDMMC 槽（引脚、总线宽度等）
    sdmmc_slot_config_t slot_config = SDMMC_SLOT_CONFIG_DEFAULT();
    slot_config.width = 4;                 // 使用 4-bit 模式（速度更快）

    // 4. 一键挂载
    esp_err_t ret = esp_vfs_fat_sdmmc_mount(
        MOUNT_POINT,      // 挂载点，之后用 /sdcard/xxx 访问文件
        &host,            // SDMMC 主机配置
        &slot_config,     // 引脚/槽配置
        &mount_config,    // FAT 挂载配置
        &card             // 输出：卡信息指针
    );

    if (ret != ESP_OK) {
        ESP_LOGE(TAG, "SD 卡挂载失败: %s", esp_err_to_name(ret));
        return;
    }

    // 5. 打印 SD 卡信息（型号、容量、速率等）
    sdmmc_card_print_info(stdout, card);
    ESP_LOGI(TAG, "SD 卡已挂载到 %s", MOUNT_POINT);
}
```

### 4.2 挂载点说明

`MOUNT_POINT = "/sdcard"` 是 VFS（Virtual File System）层的挂载点。挂载后，所有对 `/sdcard/xxx` 的文件操作都被重定向到 SD 卡的文件系统上。

```c
fopen("/sdcard/data.csv", "a");   // → SD 卡上的 data.csv（追加模式）
fopen("/sdcard/photos/img001.jpg", "w");  // → SD 卡上的 photos 文件夹
```

### 4.3 关键配置项详解

```c
esp_vfs_fat_mount_config_t mount_config = {
    .format_if_mount_failed = false,
    //   └─ true = 挂载失败就格式化（开发方便，生产环境别开）
    //   └─ false = 挂载失败不格式化，返回错误自己处理

    .max_files = 5,
    //   └─ 能同时打开多少个文件。每个文件句柄消耗约 400 字节 RAM
    //   └─ 如果有多任务同时写日志 + 读配置 + 写 CSV，按需增大

    .allocation_unit_size = 16 * 1024,
    //   └─ 分配单元大小（也叫簇大小），类似 Flash 的"页"
    //   └─ 16KB 对大文件读写性能好，但小文件浪费空间
    //   └─ 如果你的日志每条很小，改成 4096（4KB）更省空间

    .disk_status_check_enable = false,
    //   └─ true = 每次读写前检查卡是否还在（热插拔场景）
    //   └─ false = 不检查，更快
};
```

### 4.4 卸载（一定要做！）

```c
void unmount_sd_card(void)
{
    esp_vfs_fat_sdcard_unmount(MOUNT_POINT, card);
    ESP_LOGI(TAG, "SD 卡已卸载，可以安全拔出");
}
```

**为什么必须卸载？** SD 卡是顺序写入的，`fclose()` 时数据可能还在缓存里，卸载会确保所有缓存刷入 Flash。不卸载直接拔卡，轻则丢最后几秒的数据，重则损坏文件系统。

## 5. SD 卡上的文件操作（与 SPIFFS 的区别）

FAT 文件系统的文件操作 API 和 SPIFFS 完全一样——都是 POSIX 标准函数。主要区别在于 **FAT 支持目录结构**。

### 5.1 创建目录

SPIFFS 不支持 `mkdir`，但 SD 卡可以：

```c
#include <sys/stat.h>

// 创建 /sdcard/data/ 目录
struct stat st;
if (stat("/sdcard/data", &st) != 0) {   // 目录不存在
    mkdir("/sdcard/data", 0755);         // 创建
}
```

### 5.2 遍历目录

```c
#include <dirent.h>

void list_sd_card(void)
{
    DIR *dir = opendir("/sdcard");
    if (dir == NULL) {
        ESP_LOGE(TAG, "无法打开目录");
        return;
    }

    struct dirent *entry;
    while ((entry = readdir(dir)) != NULL) {
        if (entry->d_type == DT_REG) {
            printf("  文件: %s\n", entry->d_name);
        } else if (entry->d_type == DT_DIR) {
            printf("  目录: %s\n", entry->d_name);
        }
    }
    closedir(dir);
}
```

### 5.3 检查剩余空间

```c
void check_sd_space(void)
{
    FATFS *fs = NULL;
    f_getfree("0:", NULL, &fs);  // 获取 FATFS 信息
    if (fs) {
        DWORD total_sectors = (fs->n_fatent - 2) * fs->csize;
        DWORD free_sectors = (fs->free_clst) * fs->csize;
        ESP_LOGI(TAG, "SD 卡: 总 %lu KB, 可用 %lu KB",
                 total_sectors / 2, free_sectors / 2);
    }
}
```

## 6. CSV 数据记录（核心应用场景）

### 6.1 什么是 CSV

CSV（Comma-Separated Values）就是用逗号分隔的纯文本表格：

```csv
timestamp,temperature,humidity,pressure
2026-05-28 10:00:00,25.3,60.1,1013.2
2026-05-28 10:05:00,25.4,60.0,1013.1
2026-05-28 10:10:00,25.2,60.3,1013.0
```

- 第一行是**列名**（表头），后续每行是一条**记录**
- 每条记录用换行隔开，每列用逗号隔开
- 可以直接用 Excel 打开，或者用 Python 的 `pandas.read_csv()` 读取

### 6.2 完整的 CSV 数据记录代码

```c
/* ===========================================================
 * 定时记录传感器数据到 SD 卡 CSV 文件
 * =========================================================== */

#define SD_MOUNT   "/sdcard"
#define CSV_FILE   "/sdcard/data/sensor_log.csv"
#define LOG_INTERVAL_MS  (5 * 60 * 1000)  // 5 分钟

// 检查 SD 卡是否挂载（通过 stat 判断挂载点是否存在）
static bool sd_is_mounted(void)
{
    struct stat st;
    return (stat(SD_MOUNT, &st) == 0 && S_ISDIR(st.st_mode));
}

// 写入一行 CSV 数据
static esp_err_t log_to_csv(float temp, float humidity)
{
    // 获取当前时间（需要先初始化 SNTP 或 RTC）
    time_t now = time(NULL);
    struct tm timeinfo;
    localtime_r(&now, &timeinfo);
    char timestamp[32];
    strftime(timestamp, sizeof(timestamp), "%Y-%m-%d %H:%M:%S", &timeinfo);

    // 打开文件（追加模式 "a"）
    FILE *f = fopen(CSV_FILE, "a");
    if (f == NULL) {
        ESP_LOGE(TAG, "无法打开 CSV 文件");
        return ESP_FAIL;
    }

    // 检查文件是否为空，是则先写表头
    fseek(f, 0, SEEK_END);
    if (ftell(f) == 0) {
        fprintf(f, "timestamp,temperature,humidity\n");
    }

    // 写入数据行
    fprintf(f, "%s,%.1f,%.1f\n", timestamp, temp, humidity);

    fclose(f);
    return ESP_OK;
}

// 传感器记录任务（在 xTaskCreate 中运行）
void sensor_logging_task(void *pv)
{
    while (1) {
        // 1. 检查 SD 卡是否可用
        if (!sd_is_mounted()) {
            ESP_LOGW(TAG, "SD 卡未挂载，跳过本次记录");
            vTaskDelay(pdMS_TO_TICKS(10000));
            continue;
        }

        // 2. 读取传感器数据
        float temp = read_temperature();    // 你的读传感器函数
        float hum = read_humidity();

        // 3. 写入 CSV
        esp_err_t ret = log_to_csv(temp, hum);
        if (ret != ESP_OK) {
            ESP_LOGE(TAG, "写入失败");
        }

        // 4. 等待下一次记录
        vTaskDelay(pdMS_TO_TICKS(LOG_INTERVAL_MS));
    }
}
```

### 6.3 记录策略设计

| 策略 | 做法 | 好处 | 风险 |
|------|------|------|------|
| **每次写一条立即 fclose** | open → write → close | 掉电不丢数据 | 频繁 open/close 慢，磨损卡 |
| **攒一批一次性写** | 攒 10 条，open → 写 10 行 → close | 速度快、磨损小 | 攒的过程中掉电丢这批数据 |
| **一直保持打开** | open 后不 close，定时 fflush | 最快 | 文件句柄一直被占用，掉电丢更多 |

**本项目推荐：** 5 分钟记录一次 → 每次 open → write → close。频率很低，open/close 的开销可以忽略，数据安全最重要。

### 6.4 CSV 文件切分策略

日志文件不能无限大。SD 卡容量够，但单文件太大打开慢、拷出来不方便。常见做法是按天/按月切分：

```c
// 生成带日期的文件名（每天一个文件）
char csv_path[64];
time_t now = time(NULL);
struct tm *t = localtime(&now);

snprintf(csv_path, sizeof(csv_path),
         "/sdcard/data/sensor_%04d%02d%02d.csv",
         t->tm_year + 1900, t->tm_mon + 1, t->tm_mday);

// 每天自动换文件，不会出现几十 MB 的大文件
```

## 7. SDMMC vs SDSPI 详细对比

| 对比项 | SDMMC（硬件 SD 控制器） | SDSPI（通过 SPI 模拟） |
|--------|----------------------|---------------------|
| ESP-IDF 头文件 | `driver/sdmmc_host.h` | `driver/sdspi_host.h` |
| 主机定义 | `SDMMC_HOST_DEFAULT()` | `SDSPI_HOST_DEFAULT()` |
| 总线配置 | `sdmmc_slot_config_t` | `spi_bus_config_t` + CS |
| 挂载函数 | `esp_vfs_fat_sdmmc_mount()` | `esp_vfs_fat_sdspi_mount()` |
| 最大频率 | 40MHz | 20MHz |
| 总线宽度 | 1-bit 或 4-bit | 1-bit（SPI 本身特性） |
| 额外步骤 | 无 | 需先 `spi_bus_initialize()` |
| 引脚映射 | 部分引脚固定（与 GPIO 矩阵相关） | 任意 GPIO |

### SDMMC 挂载代码简化版

```c
sdmmc_host_t host = SDMMC_HOST_DEFAULT();
sdmmc_slot_config_t slot_config = SDMMC_SLOT_CONFIG_DEFAULT();
slot_config.width = 4;  // 4-bit 模式

sdmmc_card_t *card;
esp_vfs_fat_sdmmc_mount("/sdcard", &host, &slot_config, &mount_config, &card);
```

### SDSPI 挂载代码简化版

```c
// 需要额外三步：引脚定义 + SPI 总线初始化 + CS 配置
#define PIN_NUM_MISO  GPIO_NUM_2
#define PIN_NUM_MOSI  GPIO_NUM_15
#define PIN_NUM_CLK   GPIO_NUM_14
#define PIN_NUM_CS    GPIO_NUM_13

sdmmc_host_t host = SDSPI_HOST_DEFAULT();

spi_bus_config_t bus_cfg = {
    .mosi_io_num = PIN_NUM_MOSI,
    .miso_io_num = PIN_NUM_MISO,
    .sclk_io_num = PIN_NUM_CLK,
    .quadwp_io_num = -1,
    .quadhd_io_num = -1,
};
spi_bus_initialize(host.slot, &bus_cfg, SDSPI_DEFAULT_DMA);

sdspi_device_config_t slot_config = SDSPI_DEVICE_CONFIG_DEFAULT();
slot_config.gpio_cs = PIN_NUM_CS;
slot_config.host_id = host.slot;

sdmmc_card_t *card;
esp_vfs_fat_sdspi_mount("/sdcard", &host, &slot_config, &mount_config, &card);
```

## 8. 卡检测（CD）与热插拔

### 8.1 什么是卡检测（Card Detect）

很多 SD 卡座有一个额外的机械开关引脚——卡插进去时电平变化。ESP-IDF 的 `sdmmc_slot_config_t` 支持配置：

```c
slot_config.gpio_cd = GPIO_NUM_XX;  // 卡检测引脚
// 配置后，挂载时会先检查卡是否已插入
```

### 8.2 软件轮询检测

如果板子没有引出 CD 引脚，可以定时尝试 stat 挂载点：

```c
static bool check_card_present(void)
{
    // 尝试挂载检测：如果 SDMMC 初始化失败，大概率卡没插
    sdmmc_host_t host = SDMMC_HOST_DEFAULT();
    esp_err_t ret = sdmmc_card_init(&host, &card);
    if (ret != ESP_OK) {
        return false;
    }
    sdmmc_card_deinit(&host);  // 只是检测，用完释放
    return true;
}
```

更简单的方式——直接尝试访问文件：

```c
FILE *f = fopen("/sdcard/test.txt", "r");
if (f == NULL) {
    // SD 卡可能不在
} else {
    fclose(f);
}
```

## 9. 时间戳（RTC）与日志关联

数据记录如果没有时间戳，就只是一堆没用的数字。ESP-IDF 的时间源有两种：

### 9.1 使用 SNTP 获取网络时间（推荐）

```c
#include "esp_sntp.h"

void initialize_sntp(void)
{
    sntp_setoperatingmode(SNTP_OPMODE_POLL);
    sntp_setservername(0, "pool.ntp.org");
    sntp_init();

    // 等待时间同步
    time_t now = 0;
    int retry = 0;
    while (now < 100000 && retry < 10) {
        vTaskDelay(pdMS_TO_TICKS(1000));
        time(&now);
        retry++;
    }
}
```

### 9.2 使用 RTC（无网络时）

ESP32-S3 内部有 RTC，掉电后由外部 RTC 电池（如果有）或内部备份供电：

```c
// 设置初始时间（手动）
struct tm tm = {
    .tm_year = 2026 - 1900,
    .tm_mon  = 4,        // 5月
    .tm_mday = 28,
    .tm_hour = 10,
    .tm_min  = 0,
    .tm_sec  = 0,
};
time_t t = mktime(&tm);
struct timeval now = { .tv_sec = t };
settimeofday(&now, NULL);
```

实际项目中：**有 WiFi 时 SNTP 同步 → 没 WiFi 时 RTC 维持**，两者无缝切换。

## 10. 错误处理与防护

### 10.1 常见错误

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| `ESP_ERR_TIMEOUT` | SD 卡没插或接触不良 | 检查卡是否插好，引脚是否虚焊 |
| `ESP_FAIL` | FAT 文件系统损坏 | 电脑上修复或重新格式化 |
| `ESP_ERR_NOT_FOUND` | 分区表错误（极少出现） | 硬件问题，检查 SD 卡座 |
| 写文件返回 0 | SD 卡满了 | 检查剩余空间，删除旧日志 |
| 打开文件返回 NULL | 路径不对或卡未挂载 | 确认挂载路径、目录是否存在 |

### 10.2 防写入掉电损坏

```c
// 安全写入模式：先写临时文件，再 rename 覆盖
void safe_write_data(float temp, float hum)
{
    // 1. 先写 .tmp 文件
    FILE *f = fopen("/sdcard/data/tmp.csv", "w");
    if (f == NULL) return;
    fprintf(f, "%.1f,%.1f\n", temp, hum);
    fclose(f);

    // 2. rename 覆盖正式文件（原子操作）
    rename("/sdcard/data/tmp.csv", "/sdcard/data/latest.csv");
    // 此时如果掉电，最多丢 tmp.csv，latest.csv 还是完整的
}
```

### 10.3 空间监控

```c
// 记录前检查剩余空间，低于阈值时删除最早的文件
void check_space_before_write(void)
{
    FATFS *fs;
    f_getfree("0:", NULL, &fs);
    uint32_t free_kb = fs->free_clst * fs->csize / 2;

    if (free_kb < 512) {  // 剩余不到 512KB
        ESP_LOGW(TAG, "SD 卡空间不足 (%lu KB)，清理旧日志", free_kb);
        // 删除 7 天前的日志文件（按文件名日期判断）
        // ...
    }
}
```

## 11. 完整挂载+记录+卸载流程（伪代码）

```
系统启动
  │
  ├─ 初始化：SDMMC 主机
  ├─ esp_vfs_fat_sdmmc_mount("/sdcard", ...)
  │     ↓
  ├─ 成功？ → 打印 SD 卡信息（型号、容量、速率）
  │     ↓
  ├─ 创建目录（如果需要）：mkdir("/sdcard/data")
  │     ↓
  ├─ 启动传感器记录任务（每 5 分钟写 CSV）
  │     ↓
  ├─ [其他任务...]
  │     ↓
  ├─ 用户按键 → 卸载 SD 卡
  │     ↓
  └─ esp_vfs_fat_sdcard_unmount("/sdcard", card)
        → "可以安全拔出 SD 卡"
```

## 12. 初学者常见错误

| ❌ 错误 | ✅ 正确 | 原因 |
|---------|---------|------|
| 用 `/sdcard/data.csv` 但没创建 `/sdcard` | 挂载点自动存在，直接用 | FAT 挂载后 `/sdcard` 就是根目录 |
| 写文件用 `"w"` 模式 | 日志追加用 `"a"` | `"w"` 会清空旧数据 |
| 不检查 `fopen` 返回值 | 每次 `fopen` 都要检查 NULL | SD 卡可能未挂载、已拔出、空间满 |
| 忘记 `fclose` | open 和 close 配对 | 文件句柄泄漏，`max_files` 耗尽 |
| 热插拔不 unmount | 先 unmount 再拔卡 | 可能损坏文件系统，卡上的数据可能缺失 |
| 日志文件不切分 | 按日期命名文件 | 单文件几 MB 后打开慢、拷贝慢 |
