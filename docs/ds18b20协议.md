# 1-Wire 单总线协议（DS18B20）

## 概述

1-Wire 是 Dallas Semiconductor（现 Maxim）开发的单线通信协议。DS18B20 温度传感器使用该协议与主机通信。

## 物理层

- 一根数据线 DQ：双向数据输入/输出
- GPIO 配置为 **开漏模式**（`GPIO_MODE_INPUT_OUTPUT_OD`）+ 外部上拉电阻
- 空闲状态：高电平（由上拉电阻维持）
- 通过切换输出高低电平 + 读取引脚电平实现双向通信

## 通信时序

所有时序由 `esp_rom_delay_us()` 微秒级延时精确控制，**没有独立的时钟线**。

### 复位脉冲

```
主机:  ┌────────────────────┐               ┌──────
       │  拉低 ≥750μs       │  释放          │
       └────────────────────┘               ↑
                                            │
DS18B20:                         ┌──────────┘
                                 │ 拉低 200μs
                                 └──────────
```
主机拉低 750μs → 释放总线 → DS18B20 拉低 200μs 作为应答

### 写时序

```
写 0:     ┌──────────────────────────────┐   ┌────
          │  拉低 60μs                   │   │
          └──────────────────────────────┘   ↑

写 1:     ┌──┐                              ┌────
          │  │  拉高 60μs                   │
          └──┘   2μs                        ↑
```

### 读时序

```
主机:     ┌──┐                              ┌────
          │  │  释放，DS18B20控制总线       │
          └──┘   ↓                         ↑
          2μs    主机在 15μs 内采样
```

| 操作 | 时序要求 |
|------|---------|
| 写 0 | 拉低 60~120μs，然后释放 |
| 写 1 | 拉低 1~15μs，释放，然后拉高 60~120μs |
| 读位 | 拉低 1~15μs，释放，15μs 内完成采样 |

### 数据传输

- **低位优先**：读写字节时先传 LSB（bit0），后传 MSB（bit7）
- 每个位约 60~120μs，一个字节约 1ms

## 指令集

### ROM 指令（多设备寻址）

| 指令 | 代码 | 说明 |
|------|------|------|
| Search ROM | 0xF0 | 搜索总线上的设备 |
| Read ROM | 0x33 | 读取设备 64 位序列号 |
| Match ROM | 0x55 | 匹配指定序列号的设备 |
| Skip ROM | 0xCC | 跳过 ROM 寻址（单设备时使用） |
| Alarm Search | 0xEC | 搜索报警设备 |

### 功能指令

| 指令 | 代码 | 说明 |
|------|------|------|
| Convert T | 0x44 | 启动温度转换 |
| Read Scratchpad | 0xBE | 读暂存器（含温度值） |
| Write Scratchpad | 0x4E | 写暂存器 |
| Copy Scratchpad | 0x48 | 将暂存器复制到 EEPROM |
| Recall EEPROM | 0xB8 | 从 EEPROM 恢复暂存器 |

## 温度读取流程

```
┌─────────────────────┐
│  ds18b20_start()    │ 复位 + 0xCC + 0x44
│  启动温度转换        │ 等待转换完成（最大 750ms）
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  ds18b20_reset()    │ 复位
│  ds18b20_check()    │ 检测应答
│  0xCC               │ 跳过 ROM
│  0xBE               │ 读暂存器
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  读取 LSB (TL)      │ 暂存器 Byte0
│  读取 MSB (TH)      │ 暂存器 Byte1
└─────────┬───────────┘
          ▼
┌─────────────────────┐
│  temp = (TH << 8)   │
│          | TL       │
│  temp × 0.625       │ 12位分辨率，精度 0.0625°C
│  = 实际温度 × 10    │
└─────────────────────┘
```

## 温度数据格式

12 位分辨率下，温度寄存器 16 位格式：

```
Bit 15   Bit 14-11   Bit 10-0
  S         Sign      温度数据
```

- S=0：正数，直接读取
- S=1：负数，补码形式
- LSB 每 bit = 0.0625°C
- 参考代码返回 `温度 × 10`（方便显示），需除以 10 得实际温度

## 驱动 API

| 函数 | 说明 |
|------|------|
| `ds18b20_init()` | 初始化 GPIO（开漏模式），复位检测，返回 0=正常 |
| `ds18b20_reset()` | 拉低 750μs 复位总线 |
| `ds18b20_check()` | 等待应答，返回 0=存在 |
| `ds18b20_read_bit()` | 读一个位 |
| `ds18b20_read_byte()` | 读一个字节（低位优先） |
| `ds18b20_write_byte()` | 写一个字节（低位优先） |
| `ds18b20_start()` | 启动温度转换 |
| `ds18b20_get_temperature()` | 获取温度值（返回 ×10 的整数） |
